# trip_app_requirements
Полное ТЗ для платформы бронирования туристических походов. 4 роли (Гость, Путешественник, Гид, Администратор), 30+ User Stories (MoSCoW), 3 Use Case, 7 групп функциональных требований. Детальная бизнес-логика: регистрация с модерацией, создание походов, бронирование, чаты (1 чат = 1 поход, JWT), интеграция с TPay.
